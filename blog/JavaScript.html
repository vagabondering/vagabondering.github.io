  • Ruby Class VS JavaScript Function


    • A Ruby class and JavaScript function are similar in terms of both being a container for scopes of variables and methods/functions. In JS, there are two kinds of function types, literal notation and constructor function. The difference is that a constructor function can create instances like a Ruby class, whereas literal notation is used to build a single object.
    As for object instances, in order to use a class and its content in Ruby, an instance need to be created, the class itself can't be executed with the method. In JS the whole function can be executed without having to create an instance at first. Some of the other differences are syntax, naming convention and the setup of variables.

    Here's an example creating a code that has two types of variables and prints them out and creating an instance of each:

    A Ruby class:

    class ClassName
    def initialize
    @instance_variable = "instance variable"
    end

    def method_name
    @instance_variable
    puts @instance_variable
    end

    def method_name2
    local_variable = 'example'
    puts local_variable
    end

    end

    class_instance = ClassName.new
    class_instance.method_name => instance_variable
    class_instance.method_name2 => local_variable

    Javascript constructor function :

    var globalVariable = "global variable"

    function printVariables() {
    console.log(globalVariable);
    var localVariable = "local variable";
    console.log(localVariable);
    }

    printVariables() => global variable local variable

    var objectInstance = new printVariables('an object');

    You can see how each differs, Ruby using def keyword to define an object and end to enclose the code and JS using var keyword to define an object and enclosing with a curly bracket. The way local and non-local variables is different as well. Ruby uses the @ prefix to define an instance variable and @@ for global variable. JS only has local or global variables and they're determined what kind from where in the scope they're defined. A global variable has to be defined outside the function and the variable can called inside the function by its name. For a local variable, it has to be inside the function accompanied with var keyword, which means a new variable gets created. Without the var keyword, the function is going to think it's an existing variable and look for where it's defined.
